Welcome to Ruby-Go!

This project is in active development stage. Main missing features are staggering:

  • No AI so you cannot play against the program. We have a huge ambition for this part, or at least are planning to have a hell of a fun trying to build it!
  • No internet remote server ability. This can be another fun part to add. Note that if it works one day we would really like to see a game happen between 3 or 4 players, since this is an added feature that we were curious about.
  • No real GUI yet. The simplistic GUI you can access by using the links below has the convenient advantage to allow us to test the program using a mouse instead of entering all the moves in console mode... This is why it was coded.
  • Mmm... Thinking a bit, we are also missing a score counting feature, and probably a bunch of other things I cannot think of right now. Life is fun! :)

Playing with Browser GUI

Start the server

First, please start your web server before clicking on the links below.
For example, you can start it with a command like:
cd .../ruby-go/src
.../ruby.exe main_server.rb
You should see a message like: 
I, [2013-08-15T20:49:24.000000 #1372]  INFO -- : Starting the server...

Start a new game

Once the server is running, you can try any of the links below:

5x5, 2 human players (nice for tests!)
9x9, 1 player against dumb AI, handicap of 5
9x9, 2 players (to load a game and check score counting feature)
Regular 19x19 board, 2 human players

Playing in console window

We use this more for testing and debugging. For this purpose this is much easier than the browser interface.
Everything is in text mode and will give you the good old DOS feeling again... 

cd .../ruby-go/src
.../ruby.exe main_console.rb

The settings are in the script "main_console.rb" itself. For example the number of players, AI, etc.
Remember to change the logging level in logging.rb if you want to see a matrix-movie-like torrent of information about what happens in the background.
Type "help" for a list of the available commands.

Run the tests

Until an autobuild is set, these should be run manually as often as needed (especially before checking in code changes).

For the unit tests: 
cd .../ruby-go/src/test
.../ruby.exe test_all.rb
For the performance (speed) test: 
cd .../ruby-go/src/test
.../ruby.exe test_speed.rb
